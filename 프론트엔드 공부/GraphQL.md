# GraphQL이란?

API를 위한 **오픈 소스 쿼리** 언어이며 서버 측 런타임이다.
데이터 간의 관계를 정의하는 강력한 스키마를 제공한다.

## GraphQL의 장단점

**장점**

- 하나의 EndPoint로 한번의 요청으로 여러 API의 정보를 가져올 수 있다.
- 필요한 데이터만 요청 가능!
- 응답 크기를 줄일 수 있음
  <br>

**단점**

- GraphQL 스키마에서 필드를 정확하게 모두 입력해야 한다.
- Map, Object 사용 불가..

## Rest API와의 차이점

- Rest API는 특정 기능을 위해 여러번 호출이 됨
- API에 대한 유지보수가 어려울 수 있음

### 마무리

GraphQL API는 프론트엔드 개발자 입장에선
쉽고 간편한 기술이지만 백엔드에선 설정할게 많고 스키마 파일도 작성해야해서 복잡할 것 같다.
