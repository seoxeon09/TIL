# 병렬이란 무엇일까?

우선 promise에 대해 들어가기 전에 병렬의 정확한 뜻부터 알아보자.

병렬이란 여러 작업이 **같은 시간대**에 겹쳐서 진행되는 것이다.

<br>

# Promise.all

여러 Promise를 병렬로 시작하고
**전부 성공했을떄**만 결과를 배열로 받는것

주의 할 점은 하나라도 실패되면 전체 실패가 된다.
<br>

# Promise.allSettled

Promise.all과 비슷하지만 다른것인데

- 성공이든 실패든 일단 결과를 끝까지 기다림
- 그리고 성공인지 실패인지 각각의 상태를 알려줌

언제 이걸 쓰냐?

- 부가 데이터 (광고)
- 관리자 페이지 등
  위 예시들처럼 일단 불러오는게 실패하더라도 화면은 보여야할때 씀

## retry란 무엇일까?

비동기 작업 실패 시, 바로 포기하지 않고 여러번 시도하는 전략이다.

- 비동기 작업이 실패하는 원인은 다양하다.
  네트워크 상태나 환경에 따라 비동기가 실패할 수도 있다.

그래서 사전에 미리 실패 시 몇 번이나 재시도 할 것인지 설정하는거라 생각하면 된다.
