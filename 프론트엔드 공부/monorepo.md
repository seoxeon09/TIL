# 모노레포(Monorepo)

모노레포는 여러 개의 프로젝트나 패키지를 하나의 저장소(repository)에서 함께 관리하는 개발 방식이다. 프론트엔드, 백엔드, 공용 라이브러리 등을 하나의 레포지토리에 두고 개발한다.

## 특징

- 하나의 저장소에서 여러 프로젝트를 관리
- 공통 코드와 설정을 쉽게 공유 가능
- 전체 코드베이스를 한 번에 파악 가능

## 장점

- 코드 재사용성이 높음
- 프로젝트 간 의존성 관리가 쉬움
- 통합 테스트와 빌드가 간편함

## 단점

- 저장소 규모가 커질 수 있음
- 빌드 및 테스트 시간이 길어질 수 있음
- 권한 관리가 복잡해질 수 있음

## 사용 예시

- 대규모 서비스
- 여러 패키지가 긴밀하게 연관된 프로젝트
